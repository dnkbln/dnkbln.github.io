Kapitel 5. Assets

Inhaltsverzeichnis

Epsioden-Assets Menü
Assets Add

Zusammenfassung

Hier wird festgelegt, welche Daten zu einer regulären Episode gehören.

Epsioden-Assets Menü

Abbildung 5.1. Bildschirmfoto vom Episoden-Assets Bereich


Im Episoden-Assets Bereich wird angegeben welche Art von Daten standardmäßig zu einer Episode gehören sollen. Hier wird somit nicht angegeben, welche konkrete Datei in einer Episode verwendet wird, sondern welche Art von Dateien zu einer regulären Podcastfolge gehören. Bei einem Audio-Podcast wird zum Beispiel angegeben welche Audio-Formate der Podcast bereitstellt. Darüber hinaus kann definiert werden, welche zusätzlichen Dateien zum Podcast gehören, z.B. Transkripte oder Kapitelmarken. Für jedes Format und jede zusätzliche Datei muss ein Eintrag erzeugt werden.

Diese Angaben werden später bei der Erzeugung der Episoden genutzt um zu prüfen, ob alle gewünschten Daten vorhanden sind.

Achtung

Die Eingaben in diesem Tab werden immer erst nach der Betätigung des Buttons Save changes gespeichert und wirksam. Bei einem Wechsel der Tabs ohne Save changes gehen alle Eingabe verloren.

Abbildung 5.2. Episoden-Assets Add New

An dieser Stelle wird das Menü zur Erzeugung eines neuen Episoden-Assets aufgerufen.

Abbildung 5.3. Episoden-Assets Liste der Assets

Hier werden alle bereits definierten Episoden-Assets aufgelistet. Zu jedem Eintrag wird Episoden-Asset Titel, der File-Type und ob der Eintrag downloadbar ist angezeigt.

Abbildung 5.4. Episoden-Assets Liste der Assets Menü

Ist die Maus über einen Eintrag in der Episoden-Assets Liste, so werden für diesen Eintrag folgende Menüeinträge angezeigt.

Edit - Ändern des Episoden-Asset Eintrags
Activating for all existing Episodes - Wurde ein Episoden-Asset hinzugefügt und gab es bereits Episoden, so kann hiermit das Asset zu allen vorhandenen Episoden hinzugefügt werden.
Delete - Löschen des Episoden-Asset Eintrags

Abbildung 5.5. Episoden-Assets Bild

Hier wird angegeben, welches Bild im Feed für die Episode verwendet wird.

Abbildung 5.6. Episoden-Assets Kapitelmarken

Hier wird angegeben wie die Kapitelmarken behandelt werden. Es kann auf die Kapitelmarken verzichtet werden oder der Benutzer kann sie manuell verwalten.
