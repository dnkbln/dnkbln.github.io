Kapitel 5. Assets

Inhaltsverzeichnis

Assets Add

Zusammenfassung

Hier wird festgelegt, welche Daten zu einer regulären Episode gehören.

Abbildung 5.1. Bildschirmfoto vom Episoden-Assets Bereich


Im Episoden-Assets bereich wird angegeben welche Art von Daten standardmäßig zu einer Episode gehören sollen. Hier wird somit nicht angegeben, welche konkrete Datei in einer Episode verwendet wird, sondern welche Art von Dateien zu einer regulären Podcastfolge gehören. Bei einem Audio-Podcast wir zum Beispiel angegeben welche Audio-Formate der Podcast bereitstellen wird. Darüber hinaus kann definiert werden, welche zusätzlichen Dateien zum Podcast gehören, z.B. Transkripte oder Kapitelmarken. Für jedes Format und jede zusätzliche Datei muss ein Eintrag erzeugt werden.

Diese Angaben werden später bei der Erzeugung der Episoden genutzt um zu prüfen, ob alle gewünschten Daten erzeugt wurden.

Achtung

Die Eingaben in diesem Tab werden immer erst nach der Betätigung des Buttons Save changes gespeichert und wirksam. Bei einem Wechsel der Tabs ohne Save changes gehen alle Eingabe verloren.
